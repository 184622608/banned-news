### 狮航接错系统数据 机头26次向下后消失在雷达
------------------------

<div class="wysiwyg">
 【新唐人北京时间2018年11月28日讯】印尼
 <a href="http://www.ntdtv.com/xtr/gb/articlelistbytag_狮子航空.html" target="_blank">
  狮子航空
 </a>
 （Lion Air）波音737 MAX客机上月起飞不久
 <a href="http://www.ntdtv.com/xtr/gb/articlelistbytag_失事坠海.html" target="_blank">
  失事坠海
 </a>
 ，机上189人全数丧命。当局今天公布初步调查报告，指客机的自动安全系统接收错误数据，导致机头26次向下；机师由起飞一刻便与系统〝搏斗〞，但最终酿成空难。敦促狮航必须改善飞航安全并加强记录飞机维修工作。
 <br/>
 <br/>
 综合报导，这架失事的波音737 MAX客机10月29日预定从印尼首都雅加达飞往邦加槟港（Pangkal Pinang），但起飞13分钟就消失在雷达上，没多久坠毁在爪哇海。
 <br/>
 <br/>
 《纽约时报》于27日报导，飞行纪录仪数据与调查方向一致，即波音公司的新操控系统〝机动特性增强系统〞（MCAS），或是涉事737 MAX 8客机坠毁的关键。
 <br/>
 <br/>
 报导说，当机头升得太高致失速时，系统就强迫机头向下。但系统疑接收错误感应器数据，令机头多次向下；机师却按旧系统程序把操控杆拉后，但最终客机仍高速坠海。
 <br/>
 <br/>
 这份报告犹如外界预料，仅提到机师事发时多次要修正客机的反失速系统，但没有解释真正出现意外的原因。最终调查报告应需到明年才会公布。
 <br/>
 <br/>
 印尼国家运输安全委员会（Indonesian National Transportation Safety Committee）的官员批评，尽管狮航过去多次都没有解决这架客机的空速表问题，却仍让客机载客飞行。要求该公司改善安全文化及更好地为修理工作做记录。
 <br/>
 <br/>
 对这次空难目前尚有几个疑团未解开，例如为何感应器出问题，客机仍获准飞行，而驾驶室的录音内容未修复，难以得知机师有否依照正确程序。另有指客机坠海前，机师把控制权交予副机师。
 <br/>
 <br/>
 前波音工程师莱梅（Peter Lemme）形容，当时就像一个〝致命捉人游戏〞，机师每隔5秒就要手动将机头移向上，但机师也未按正确程序将飞机拨乱反正。
 <br/>
 <br/>
 他说，感应器数据是否正确并不容易查出，机师事前未告知之前的航程是否出现这个问题，狮航亦没有修理好。他强调若修理好，就不会出现这次意外。
 <br/>
 <br/>
